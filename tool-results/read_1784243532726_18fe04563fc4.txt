    30→## 1. Общее описание системы
    31→
    32→**VSM ОМиК** — корпоративная информационная система для отдела мобилизации и координации персонала строительной компании «ВелесстройМонтаж». Система обеспечивает:
    33→
    34→- **Управление базой данных сотрудников** (113,000+ записей из 1С)
    35→- **Учёт затрат на авиа- и ЖД-билеты** (организации ВСП и СК)
    36→- **Ежедневный учёт персонала** (на объекте, в командировке, в отпуске)
    37→- **Календарь прилёта-вылета** (отслеживание перемещений)
    38→- **Автоматизированную обработку данных** (24 Python-утилиты)
    39→- **Файловый менеджер** (загрузка, просмотр, удаление, каталогизация)
    40→- **Систему оценок KPI** (рейтинг персонала)
    41→- **Генерацию отчётов** (по трудоустройству, увольнениям, мобилизации)
    42→
    43→Система построена как **одностраничное приложение (SPA)** с боковой навигацией и динамической загрузкой панелей. Все данные хранятся в памяти сервера (in-memory cache) и/или в SQLite через Prisma ORM.

   110→### 2.3. Ключевые архитектурные решения
   111→
   112→1. **In-memory кеш вместо полной БД** — Основная база сотрудников (113K+ записей) хранится в оперативной памяти через `globalThis` для переживания HMR. SQLite используется только для метаданных файлов.
   113→
   114→2. **JSON+base64 вместо multipart/form-data** — Caddy-прокси ломает multipart-запросы, поэтому все загрузки файлов используют JSON с base64-кодированием.
   115→
   116→3. **Динамический импорт панелей** — Все панели загружаются через `next/dynamic` с `ssr: false` для уменьшения размера начального бандла.
   117→
   118→4. **Декомиссия FastAPI** — Изначально система имела Python FastAPI бэкенд на порту 3031. Вся логика перенесена в `excel-server.ts` (Node.js), Python-бэкенд не запускается.
   119→
   120→5. **Worker Thread для тяжёлых операций** — Загрузка основной базы данных вынесена в Worker Thread для предотвращения блокировки event loop.

   431→### 5.2. useTaskStore (менеджер задач)
   432→
   433→**Файл:** `src/store/task-store.ts`
   434→
   435→Продвинутый менеджер задач с Map-хранилищем. Поддерживает:
   436→
   437→- **Типы задач:** upload, download, processing, export, load, validate, other
   438→- **Под-шаги:** Каждая задача может иметь массив `TaskStep[]` с отдельными статусами
   439→- **AbortController:** Каждая задача имеет сигнал отмены для fetch-запросов
   440→- **Auto-dismiss:** Завершённые задачи автоматически удаляются через N мс
   441→- **Автоочистка:** Каждые 30 секунд удаляются задачи, завершённые более 10 секунд назад
   442→
   443→```typescript
   444→interface TaskInfo {
   445→  id: string                    // "task-{timestamp}-{counter}"
   446→  name: string                  // Читаемое имя задачи
   447→  type: TaskInfo['type']        // Тип для группировки
   448→  sourceTab: string             // Вкладка-источник
   449→  status: TaskStatus            // pending | running | done | error | cancelled
   450→  progress: number              // 0-100, -1 = неопределённый
   451→  message: string               // Детальный статус
   452→  steps: TaskStep[]             // Под-шаги
   453→  fileName?: string             // Имя файла (если применимо)
   454→  startedAt: number             // Timestamp начала
   455→  finishedAt?: number           // Timestamp завершения
   456→  abortController?: AbortController  // Для отмены
   457→  autoDismissMs: number         // Автоудаление через N мс (default: 5000)
   458→  error?: string                // Сообщение об ошибке
   459→}
   460→```

   974→## 15. Известные проблемы и ограничения
   975→
   976→### Критические
   977→1. **502 Bad Gateway при загрузке** — Долгая обработка XLSX.read() блокирует event loop
   978→2. **Caddy ломает multipart/form-data** — Обход через JSON+base64 (на 33% больше размер)
   979→
   980→### Производительность
   981→3. **113K+ записей в памяти** — ~100+ МБ RAM
   982→4. **Нет виртуализации таблиц** — Пагинация без виртуализации
   983→5. **SheetJS segfault** — На больших файлах в Turbopack
   984→
   985→### Функциональные
   986→6. **Нет аутентификации** — NextAuth.js не настроен
   987→7. **Mock-данные** — DailyTracking, Ratings, Dashboard stats
   988→8. **10 из 19 вкладок в разработке**
   989→9. **Нет кнопки «Очистить»** — Для загрузок DailyTracking
   990→10. **Нет persisted state** — Zustand stores не персистятся
