# Описание проекта — HR-аналитический дашборд ВелесстройМонтаж

Веб-приложение для загрузки кадровых Excel-файлов, хранения данных в локальной базе SQLite и интерактивной аналитики по сотрудникам, площадкам, визам, динамике кадровых событий и расхождениям должностей.

---

## Назначение

Система объединяет несколько источников HR-данных компании:

- **Основная база 1С** (варианты М и АО) — штатная структура, даты приёма/увольнения, паспортные и кадровые поля
- **Ежедневный учёт** — актуальное состояние сотрудников на площадках (регион, виза, фактическая должность, дата прибытия)
- **Кадровые события** — приёмы, переводы, увольнения за период
- **Справочники** — классификация должностей, площадки/регионы, унификация подразделений
- **Фактические должности** — сопоставление мастер- и фактических данных
- **Оценки** — полугодовые оценки сотрудников

На основе этих данных строятся KPI, графики, таблицы, карточки сотрудников и материализованные отчёты «Хронология» и «Карточки».

## Источники данных и категории загрузки

### Справочники
- `Классификация_по_должности.xlsx` — ИТР / РОП / АУП
- `Площадки_Регион.xlsx` — унификация площадок и привязка к регионам
- `Подразделение_Участки_База_вариант_М.xlsx` / `_АО.xlsx`

### Кадровые файлы
- **Приём**, **Перевод**, **Увольнение** — события для хронологии
- **БАЗА** (вариант М или АО) — основной реестр сотрудников (~100k+ записей)
- **Ежедневный учёт** — срез на дату (~14k записей ВСМ)

### Прочее
- Фактические должности (ВСМ / СК)
- Полугодовые **Оценки** (2024–2028)

Тип импорта определяется **категорией** в интерфейсе «Файлы данных», а не именем файла. Пользователь выбирает файл через «Обзор» и загружает в нужную категорию.

## Бизнес-правила (ключевые)

### Компания по табельному номеру

| Префикс | Компания |
|---------|----------|
| ВСМ, ВМ | ВелесстройМонтаж |
| ВС | Велесстрой |
| ВУ | Велесстрой-СМУ |
| ГК | ГКК |
| СА | СМК |
| МВ | Стройконстракшен |

### Статус «Действующий»
Сотрудник считается действующим, если дата увольнения пуста **или** позже текущей даты.

### База 1С в карточке
Блок «Из базы 1С» показывается только для табельных номеров компаний **ВелесстройМонтаж** (ВМ/ВСМ) и **Стройконстракшен** (МВ).

### Формат дат
Во всём проекте канонический формат: **ДД.ММ.ГГГГ**. Импорт нормализует Excel/ISO-даты; отображение поддерживает и legacy-записи в БД.

### Хронология и KPI
- Отчёты строятся кнопкой **«Создать отчёты»** на вкладке «Файлы данных»
- Данные сохраняются в `MaterializedReportRow` для быстрого открытия
- События дедуплицируются перед подсчётом «Событий» и «Переводов» в карточке

## Установка и запуск

### Требования
- Node.js 20+
- npm

### Первичная настройка

```powershell
npm install
npm run setup
```

Скрипт `setup` создаёт `.env` из `.env.example` и применяет схему Prisma.

### Разработка

```powershell
npm run dev
```

Приложение: http://localhost:3000

### Production-сборка

```powershell
npm run build
npm start
```

Остановка сервера:

```powershell
npm run stop
```

Если сборка падает с `EBUSY` — сначала остановите сервер (`npm run stop`).

### Переносимый standalone с базой

```powershell
npm run build:deploy
```

Копирует SQLite в `.next/standalone` (~3+ GB). Обычный `npm run build` базу не копирует — `npm start` читает `dev.db` из корня проекта.

## Типовой рабочий процесс

1. **Загрузить справочники** (классификация, площадки, подразделения)
2. **Загрузить БАЗУ** (М или АО) → при необходимости нажать «Активировать»
3. **Загрузить кадровые события** (Приём, Перевод, Увольнение)
4. **Загрузить Ежедневный учёт** на нужную дату
5. Нажать **«Создать отчёты»** — материализация Хронологии и Карточек
6. Работать с вкладками **Обзор**, **Хронология**, **Сотрудники** и др.
7. При необходимости — **экспорт PDF/XLSX/CSV**

После импорта аналитические вкладки обновляются автоматически (событие `dashboard:refresh`).

## Ограничения и особенности

- **Лимит загрузки файлов:** 50 МБ (`MAX_UPLOAD_BYTES`)
- **SQLite:** большие `IN`-запросы выполняются батчами (лимит ~999 параметров)
- **Windows:** все скрипты запуска адаптированы под PowerShell
- **Legacy Python:** логика хронологии перенесена из `Files/путь_сотрудника_v4-1.py`, `v5.py`, `v5_2.py`
